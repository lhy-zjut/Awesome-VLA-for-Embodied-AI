# 基于视觉-语言-动作模型的具身智能研究进展

## 简介

具身智能（Embodied AI）旨在让智能体通过与物理世界的持续交互，实现从**感知**、**理解**到**执行**的闭环。近年来，视觉-语言-动作模型（Vision-Language-Action, VLA）逐渐成为这一方向的重要技术路线。VLA 以视觉观测与自然语言指令为输入，直接或间接生成可执行动作，从而将大模型的开放世界理解能力扩展到机器人操作、导航、自动驾驶与多机器人系统等场景。

本文档从**发展历程**、**关键模块**、**系统架构分类**、**训练与评估**、**挑战与未来方向**等角度，对基于 VLA 的具身智能研究进行结构化梳理。

## 发展历程

### 1. 早期探索

这一阶段的核心是打通视觉、语言与控制之间的接口。

- **视觉编码**：ResNet、ViT、SAM、MAE、DINOv2 等模型逐步推动视觉表征从像素级识别走向语义级理解。
- **语言建模**：BERT、GPT 系列奠定了复杂指令解析与生成基础。
- **多模态桥接**：CLIP、BLIP-2、LLaVA 等模型实现视觉与语言对齐，但尚未直接进入动作生成。
- **早期具身尝试**：CLIPort、SayCan、PaLM-E 等开始把语义推理引入机器人规划与控制环节。

### 2. 中期发展

这一阶段 VLA 开始真正形成统一框架。

- 代表性工作如 **RT-1、RT-2、VIMA、ACT、Diffusion Policy** 等。
- 核心变化是将“图像、文本、状态、动作”统一为序列或词元，在同一模型内进行端到端学习。
- 动作表征开始系统化，出现离散动作词元、连续回归、扩散生成等多种技术路线。

### 3. 落地应用

自 2024 年以来，VLA 研究明显从“通用框架构建”转向“真实部署与领域适配”。

- **推理增强**：引入链式思维、显式规划、可供性预测等机制。
- **领域扩展**：导航、操作、自动驾驶、人形机器人等方向开始形成专用 VLA 方案。
- **部署优化**：模型压缩、参数高效微调、双系统结构、世界模型辅助推理等逐渐成为热点。

### 发展阶段概览

| 阶段 | 关键词 | 代表方向 |
| :-- | :-- | :-- |
| 早期探索 | 视觉/语言/控制分离 | CLIP、BLIP-2、CLIPort、SayCan、PaLM-E |
| 中期发展 | 统一词元空间、端到端建模 | RT-1、RT-2、VIMA、ACT、Diffusion Policy |
| 落地应用 | 领域适配、推理增强、部署优化 | OpenVLA、pi-0、GR00T N1、SpatialVLA、ReKep |

## VLA 关键技术模块

### 视觉特征编码

视觉编码是 VLA 的基础组件，负责从图像或视频中提取稳定、可迁移且具语义可解释性的表征。

主要技术路线包括：

1. **预训练视觉模型迁移**：如 CLIP、DINOv2、OpenCLIP、EVA-CLIP。
2. **视觉信息词元化**：如 VQ-VAE、VQ-GAN、Q-Former、TokenLearner。
3. **三维时空表征融合**：如深度图、点云、占据网格、NeRF、3D Gaussian 等。

### 语言特征编码

语言模块不只是“理解指令”，还承担跨模态对齐与任务约束建模作用。

- 常用骨干：T5、LLaMA、Qwen、Phi 等。
- 常见交互方式：FiLM、跨注意力、Perceiver Resampler、Q-Former。
- 新趋势：语言从“指令载体”进一步演化为“策略推理接口”。

### 其它模态特征编码

为了提高 VLA 在复杂物理世界中的感知能力，研究逐步将更多模态纳入统一框架：

- **点云**：PointNet 系列、Uni3D ViT 等。
- **音频**：频谱图、SpeechTokenizer、RVQ-VAE 等。
- **触觉/力觉**：DIGIT、GelStereo 2.0、视觉式触觉编码器等。

### 上下文状态编码

除视觉与语言外，机器人自身状态同样关键，包括：

- 关节位置 / 速度 / 力矩
- 触觉与受力信息
- 里程计 / LiDAR / IMU 等

这些信息通常与视觉、语言词元共同组织为统一上下文序列，以支持闭环控制。

### 动作词元与解码

动作词元是 VLA 中最核心的桥梁，它把语义理解和物理执行连接起来。

#### 动作词元常见类型

- **语言描述**：自然语言形式的动作步骤
- **代码表征**：程序、API 调用、伪代码
- **可供性表征**：关键点、边界框、热图、分割掩码
- **运动轨迹**：状态序列或轨迹点
- **目标状态**：未来图像、点云或目标场景状态
- **推理过程**：显式链式思维或中间推理文本

#### 动作解码方式对比

| 建模方式 | 优势 | 劣势 | 适用场景 |
| :-- | :-- | :-- | :-- |
| 离散化词元 | 泛化性强，便于零样本迁移，接口统一 | 有量化误差，长序列降低控制频率 | 指令跟随、移动导航 |
| 连续回归 | 输出平滑，精度高，推理高效 | 难刻画复杂动作分布，泛化偏弱 | 精确轨迹跟踪、连续控制 |
| 扩散模型 | 能建模复杂动作分布，鲁棒性强 | 推理较慢，计算开销大 | 精细抓取、复杂装配 |
| 流匹配 | 在表达能力与效率间更均衡 | 对训练数据质量要求高 | 需要实时性的操作任务 |

### 控制策略

VLA 控制策略通常分为两个层面：

#### 1. 任务规划模块

负责把抽象语义目标转成可执行的步骤或子任务。

- 代表方向：PaLM-E、EmbodiedGPT、LEO、SayCan、ProgPrompt、CaP。
- 主要目标：任务分解、在线重规划、可行性评估、空间推理。

#### 2. 底层控制策略

负责将高层目标映射为具体控制量。

- **扩散策略**：如 Diffusion Policy、DP3、MDT、RDT-1B
- **自回归策略**：如 RT-1、RT-2、OpenVLA、UniVLA

两者的核心差异在于：扩散更擅长复杂动作分布与平滑控制，自回归更适合高效推理与多模态统一建模。

## VLA 系统架构分类

本文将现有 VLA 系统归纳为三类：**单系统**、**双系统**、**层次化系统**。

### 架构总览

| 架构类型 | 核心特征 | 优势 | 局限 |
| :-- | :-- | :-- | :-- |
| 单系统 | 感知-理解-控制在同一模型中统一完成 | 端到端优化强，结构简洁 | 实时性和部署压力大 |
| 双系统 | 慢推理系统 + 快控制系统并行协作 | 兼顾语义推理与高频控制 | 接口设计复杂，协同训练困难 |
| 层次化系统 | 通过代码/关键点/子任务等中间表征连接规划与执行 | 可解释性强，便于调试与验证 | 中间表征设计难，误差可能逐级放大 |

### 单系统 VLA

单系统 VLA 将感知、语义理解和动作生成整合进同一计算图中，强调端到端建模与统一优化。

**代表性方向：**
- RT-1 / RT-2 / RT-2-X
- OpenVLA
- RDT-1B
- CogACT
- SpatialVLA

**特点：**
- 统一序列建模
- 便于研究动作词元与多模态对齐机制
- 更适合分析规模法则与跨任务泛化规律

### 双系统 VLA

双系统 VLA 显式拆分“慢语义推理”和“快物理控制”。

- **System 2（慢系统）**：负责语义解析、场景理解、长程规划
- **System 1（快系统）**：负责高频闭环控制与局部动作执行

**代表性方向：**
- pi-0 / pi-0.5 / pi0-FAST
- GR00T N1
- ChatVLA
- Diffusion-VLA
- SmolVLA
- GR-3

**特点：**
- 更接近“慢思考 + 快反应”的认知分工
- 适合动态环境、交互式导航、高频控制任务
- 但跨系统接口容易成为新的性能瓶颈

### 层次化系统 VLA

层次化系统通过显式中间表征连接高层规划与低层控制，强调可解释性与模块化。

常见中间表征形态：

- **程序化表征**：程序、伪代码、API 调用
- **关键点表征**：关键点、路径点、可供性约束
- **子任务序列表征**：自然语言子目标、技能链

**代表性方向：**
- PaLM-E
- ReKep
- Instruct2Act
- RoboMatrix
- HiRobot
- DexVLA
- RT-H

**特点：**
- 决策过程更透明
- 便于人工干预、系统集成与形式化验证
- 更适合工业、医疗等安全要求高的场景

## 训练与评估

### 训练数据

当前主流 VLA 采用一种“具身特异性递增、数据规模递减”的分层多源训练范式：

| 层级 | 数据来源 | 作用 |
| :-- | :-- | :-- |
| 底层 | 网页图文、人类视频 | 视觉-语言对齐、时序语义先验 |
| 中层 | 仿真与合成数据 | 技能组合、任务结构、低成本扩展 |
| 顶层 | 真实机器人演示 | 物理可行性、接触真实性、跨本体适配 |

**代表性数据来源：**
- 底层：Ego4D、EPIC-KITCHENS-100、Something-Something V2
- 中层：MimicGen、DexMimicGen、RoboCasa、robosuite、Habitat、Isaac Gym
- 顶层：真实机器人轨迹、力觉/音频多模态演示、长程任务专用数据

### 预训练

预训练大致可分为三种路线：

1. **单一领域数据训练**
   - 先在大规模机器人轨迹上训练，再在目标任务数据上微调
   - 代表：Octo、RDT

2. **分阶段跨领域训练**
   - 先做互联网级视觉语言对齐，再用机器人数据注入操控能力
   - 代表：RT-2、OpenVLA、GR-1、EgoVLA

3. **联合多任务跨领域训练**
   - 在统一模型中协同学习多源异构数据
   - 代表：GATO、UP-VLA、ChatVLA 系列

### 后训练

#### 1. 自监督对齐

- 利用视觉-文本对齐减少动作标注依赖
- 引入弱监督视频信号和场景增强，提高复杂环境鲁棒性

#### 2. 参数高效监督微调

- 常用方法：LoRA、适配器、量化、剪枝、动作压缩
- 目标：在少量任务数据下完成快速适配，同时保持通用能力

#### 3. 强化学习微调

主要包括三条路线：

- **端到端强化学习微调**：如 ConRFT、VLA-RL、ReWiND、iRe-VLA
- **强化学习底层策略优化**：如 Humanoid-VLA、NaVILA、SLIM、DexTOG
- **强化学习用于数据生成**：利用 RL 生成高质量轨迹，再进行监督蒸馏

### 仿真环境与基准测试

仿真环境在 VLA 研究中仍然不可替代，主要用于：

- 大规模低成本数据生成
- 多任务可控训练
- 自动化评估与调参
- 仿真到现实迁移研究

#### 代表性仿真环境 / 基准

### 长程任务评估的开放性挑战

对长程任务而言，仅看最终成功率远远不够，还需要关注：

1. **任务成功粒度与部分奖励设计**
2. **鲁棒性与错误恢复能力**
3. **组合泛化与指令遵循能力**
4. **仿真到现实的评估鸿沟**

未来评估体系应同时覆盖：
- 任务成功率
- 执行效率（Hz / 时延）
- 安全性（碰撞、越界、力超限）
- 恢复能力（恢复成功率、恢复时间）

## 存在挑战

### 1. 实时推理与计算效率挑战

- 自回归推理速度常只有数 Hz，难满足 >100 Hz 控制需求
- Transformer 计算复杂度高，视觉词元成本大
- 边缘设备部署受限于显存、带宽和功耗
- 并行解码虽可提速，但可能牺牲轨迹平滑性与精度

### 2. 泛化能力与跨场景迁移障碍

- 高质量动作标注数据稀缺
- 跨任务、跨环境、跨机器人本体迁移困难
- 连续动作空间缺少离散符号空间那样清晰的语义边界
- 模型更依赖统计相关性而非物理因果规律

### 3. 环境适应与安全保障难题

- 复杂光照、遮挡、动态背景会显著影响感知性能
- 模糊语言指令仍是困难点
- 安全机制目前很多仍停留在阈值式被动防护
- 对抗补丁、后门攻击等安全威胁已开始显现

### 4. 评估体系与伦理问题

- 现有基准多集中在结构化环境，难覆盖开放世界风险
- 数据偏见可能被模型继承并放大到物理行为中
- 责任归属、决策溯源、可解释性仍缺乏统一规范
- 公平性、安全性与价值对齐将成为重要长期议题

## 应用场景

### 1. 人形机器人平台

- 高层语义理解 + 高频全身控制
- 面向家庭、服务、开放环境操作

### 2. 自动驾驶系统

- 多视角视觉 + 车载传感器 + 自然语言统一建模
- 直接生成轨迹或控制量，提升端到端决策能力

### 3. 工业制造机器人

- 面向非标零件装配、物流分拣、人机协作
- 支持自然语言任务重配置与少样本快速适配

### 4. 医疗护理与手术机器人

- 辅助手术操作、护理协作、康复支持
- 对精度、安全和可解释性要求极高

### 5. 农业机器人

- 作物监测、病虫害识别、精准施药、选择性采摘
- 适合多传感器、多季节、多环境泛化研究

## 未来展望

### 1. 从被动感知走向主动推理

未来 VLA 需要从“看到什么就做什么”走向“理解为什么这样做、预测后果再执行”。

关键路径包括：
- 世界模型嵌入
- 多步滚动预测
- 因果发现与主动干预
- 模型预测控制与策略蒸馏结合

### 2. 实现持续学习与终身适应

未来系统不应只在离线训练后冻结，而应具备：
- 在线适应新任务/新物体/新环境的能力
- 参数高效更新能力
- 情景记忆与经验回放机制
- 基于人类反馈与自监督信号的持续学习能力

### 3. 深化应用场景与系统集成

VLA 的真正价值在于和现实系统深度融合：
- 多平台通用技能迁移
- 人机混合协作
- 多机器人协同
- 与经典建图、控制、安全模块协同工作

### 4. 构建可信赖的负责任 VLA 生态系统

未来不仅要追求性能，还要建立：
- 决策可解释与日志追踪机制
- 安全对齐与形式化验证方法
- 公平性与偏见审计工具
- 面向研发、评估、部署全流程的责任框架
